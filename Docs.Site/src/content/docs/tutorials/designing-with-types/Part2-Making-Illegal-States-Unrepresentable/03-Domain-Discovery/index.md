---
title: "도메인 발견"
---

## 개요

Union type 리팩터링 과정에서 `ContactMethod`라는 숨겨진 도메인 개념이 드러납니다. 타입이 도메인 통찰을 이끌어내는 과정을 체험합니다.

## 학습 목표

1. 타입 리팩터링이 도메인 개념을 발견하게 하는 과정 이해
2. ContactMethod 추출을 통한 도메인 모델 개선
3. 패턴 매칭과 확장성

## 핵심 통찰

ContactInfo를 union type으로 바꾸는 과정에서, "연락 방법(ContactMethod)"이라는 **새로운 도메인 개념이** 발견됩니다. 이것은 원래 코드에서는 보이지 않던 비즈니스 개념입니다.

## 요약

- 타입 리팩터링은 단순한 코드 정리가 아닙니다
- 타입 구조가 도메인 개념을 드러내고, 비즈니스 통찰로 이어집니다
- 새 ContactMethod 추가 시 컴파일러가 처리 누락을 알려줍니다
