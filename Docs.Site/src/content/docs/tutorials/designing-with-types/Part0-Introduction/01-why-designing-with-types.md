## 타입 주도 설계란

타입 주도 설계(Type-Driven Design)는 도메인 규칙을 타입 시스템에 인코딩하여, **잘못된 상태가 표현 자체가 불가능하도록** 만드는 설계 접근법입니다.

전통적인 방법에서는 비즈니스 규칙을 if 문과 유효성 검사로 보호합니다. 타입 주도 설계에서는 규칙을 타입 구조 자체에 녹여 넣어, **컴파일러가 규칙 위반을 방지**합니다.

## 왜 지금 이것이 필요한가?

C# 14의 언어 기능이 성숙하면서, 이전에는 F#에서만 자연스러웠던 타입 주도 설계가 C#에서도 실용적으로 가능해졌습니다.

| C# 기능 | 활용 |
|---------|------|
| `record` / `sealed record` | 불변 데이터, 값 동등성, 판별 공용체 시뮬레이션 |
| `switch` 식 + 패턴 매칭 | 완전성 검사(exhaustiveness check) |
| `required` 속성 | 필수 필드 강제 |
| `Fin<T>`, `Validation<Error, T>` | 실패 가능한 생성을 타입으로 표현 |

## functional-valueobject와의 관계

[함수형 값 객체 구현](../../functional-valueobject/) 튜토리얼에서 `SimpleValueObject<T>`, `ValidationRules`, `DomainError` 같은 **구현 도구를** 익혔다면, 이 튜토리얼에서는 그 도구들을 **언제, 왜 사용하는지** 설계 관점에서 다룹니다.

| | 함수형 값 객체 구현 | 타입으로 도메인 설계하기 |
|---|---|---|
| **질문** | 안전한 값 객체를 어떻게 만드는가? | 왜 이 타입 구조가 필요한가? |
| **새로운 패턴** | — | Union types, State machines |

## 원문 소개

이 튜토리얼은 Scott Wlaschin의 F# 시리즈 "Designing with Types"(8편)를 기반으로 합니다. 원문의 핵심 아이디어를 유지하면서, C# 14와 Functorium 프레임워크의 패턴으로 재구성했습니다.
