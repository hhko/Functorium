title: "쇼핑 카트 라이프사이클"
---

## 개요

Empty → Active → Paid 완전한 상태 기계를 구현합니다. 상태 건너뛰기가 타입으로 불가능합니다.

## 학습 목표

1. 완전한 상태 기계 설계
2. 상태별 데이터와 전이 함수 구현
3. "상태 건너뛰기 불가"를 타입으로 보장

## 상태 전이 다이어그램

```
Empty → Active → Paid
  ↑       ↓
  └───────┘ (아이템 모두 제거)
```

## 요약

- 상태 기계의 모든 상태를 sealed record로 정의합니다
- 전이 함수가 유효한 전이만 허용합니다
- Empty에서 Paid로 직접 전이할 수 없습니다 — 타입이 이를 강제합니다
